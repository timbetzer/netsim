Die srcip- und dstip-Header können leicht geändert werden, um unterschiedliche Quell- und Zieladressen festzulegen. In den vorherigen Levels haben wir gesehen, dass sich durch Ändern der Zieladresse die Richtung ändert, in die ein Paket gesendet wird. Beachte jedoch, dass wir keine Quelladresse angeben mussten. Tatsächlich berücksichtigen Router nicht, woher ein Paket kommt, sondern nur, wohin es geht. Eine Sicherheitslücke des Internets besteht darin, dass wir ein Paket leicht so aussehen lassen können, als käme es von einer bestimmten Adresse, obwohl es in Wirklichkeit von einer anderen stammt. Das nennt man Spoofing.

Um dieses Level zu bestehen, sende ein Paket von Alices Computer zu Bobs Computer, das so aussieht, als käme es von Carol.

Beachte, dass wir Bob glauben lassen können, dass Carol mit ihm spricht, obwohl wir keine Pakete von Carol senden dürfen!
